Testing if num is an Integer

All numbers in MATLAB are by default double, not int. Therefore, checking if a variable value is an integer is not completely straight forward. Firstly, if the class of num is a double, then the only way to verify if it is in an integer is indirectly. A property of all integers is that they are finite, so while inf, NaN etc. are double, they are not finite; therefore, checking if a double is finite can allow for filtering out inf, NaN etc. However, integers are only single whole numbers, not arrays, so the length of the result of checking if num is finite must also be 1. Lastly, a property of integers if that if floor division is performed on them, the result will equal the input. Therefore, if num is a double, checking the three described properties will determine if num is an integer.

However, if num's class is an int class, the function should still output that num is an int. However, MATLAB has multiple classes of ints, yet the first three characters in all the int classes are 'int'. Therefore, by checking the condition if the first three charactersof num's class are 'int', it can be determined if num is an int.
