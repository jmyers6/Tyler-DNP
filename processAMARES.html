This function is similar to loadJmruiAmares, but allows for keeping related AMARES files in together in one output. Instead of a 1x1 structure array being generated, a 1xnumber of files size structure array is generated instead. This function also allows for manually entering in the files to be concatenated if necessary.

Input

inp: is a cell containing each .txt AMARES output file in a separate element. In the case of only processing one file, inp is a 1x1 cell, where the sole element is the file to be processed

Output

fitted: is a structure array of the files stored in inp

Child Functions

loadJmruiAmares: parses the .txt output from AMARES in JMRUI into a structure array

validateAMARES: verifies that a file can be passed into loadJmruiAmares without throwing an exception
